# BENCH — coach-mcp 1.0.10, 3-leaf swarm

**Date:** 2026-05-29
**Coach:** `@aibuilders/mcp-coach-server@1.0.10` (released 2026-05-27)
**Spec size:** ~130 KB (~3,553 lines, full OpenAPI)
**Workload:** 3 leaves each consuming an OpenAPI spec slice
**Script:** `scripts/bench_coach_swarm.py`
**Raw JSON:** [`2026-05-29-coach-1.0.10-3leaf.json`](./2026-05-29-coach-1.0.10-3leaf.json)

## Modes

- **naive** — each leaf spawns its own MCP child process and calls
  `get_api_specification`. Full spec round-trips through MCP stdio
  N times to the swarm orchestrator (= what an LLM agent pays in tokens).
- **hubcache** — single `CoachCache.fetch_spec()` at hub via raw httpx;
  leaves consume in-memory dict slice; MCP never touched.

## Results

| Mode | Pass | Wall (ms) | Bytes to orchestrator | MCP calls |
|---|---|---:|---:|---:|
| naive    | cold | 739.2 | 391,371 | 3 |
| hubcache | cold | 439.6 | **94**  | 0 |
| naive    | warm | 252.5 | 391,368 | 3 |
| hubcache | warm | **1.4** | **94** | 0 |

## Headline

- Warm hub-cache is **180× faster wall** + **4,164× smaller payload** vs warm naive.
- 1.0.10's new disk cache helps `naive` warm (252 ms vs cold 739 ms = 2.9× speedup)
  — addresses upstream 429 risk + repeat network fetch — but does not reduce the
  payload returned to the swarm orchestrator. Each leaf still pulls full spec
  through MCP stdio.
- Hub-cache invariant (ADR-0004) still decisive on both axes; remains correct
  shape regardless of MCP-server-side caching.

## Side effect — bug surfaced

Bench run uncovered a real defect in `agentdex.coach.CoachCache`:
constructor accepts a `cache_dir` argument but ignored it, using
module-level `SPEC_CACHE_PATH` instead. The hub-fanout unit test passed
only because it used `FakeCache`, so the live path was never exercised
before bench.

**Fix:** moved spec/etag paths to `self.spec_path` / `self.etag_path`
computed from `cache_dir`. Committed in same patch as the bench.

## Recommendations

1. Keep hub-cache invariant in agentdex production swarms.
2. Adopt granular tools when 鸭哥 ships them — will allow leaves to skip
   the hub-cache step for endpoint-specific lookups (smaller per-leaf
   payload than slicing a 130 KB blob in-memory).
3. For agentdex consumers running LLM-driven leaves, expose only the
   leaf's slice key + result (94 bytes), never the spec itself.
